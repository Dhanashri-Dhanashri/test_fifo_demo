Problem Statement

Why this project matters for functional safety closure

Challenge

Fault simulation logs are large and status-heavy. Translating them into clear safety actions is slow without automation.

Risk

Latent-fault hotspots remain hidden, causing mitigation effort to drift to low-impact areas and delaying safety signoff.

Project Goal

Produce repeatable, evidence-backed outputs: fault class vs mechanism mapping, block-level diagnostic coverage, and latent risk indicators.

Detection Mechanisms

CPU redundancy, ECC memory protection, and interconnect monitoring intent

CPU Redundancy Compare

Duplicate control paths detect mismatches in flags and pointer states, generating immediate error signals.

Observed campaign DC: 90.79% for redundant_compare classed faults.

ECC Memory Protection

ECC encode/decode with syndrome-based detection secures data path integrity in dual-port RAM access.

Observed campaign DC: 23.44% for ecc_syndrome mapped faults.

Interconnect Monitoring

Functional signal checks (ReadEn/WriteEn/Clock/Data) and rule-based mapping support interface-level anomaly detection.

Current run emphasis: control + memory dominant fault exposure.
