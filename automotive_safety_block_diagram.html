FIFO Functional Safety Block Diagram

Automotive presentation theme - design architecture, fault injection flow, and safety coverage outputs

1) Complete Design Architecture

Testbench `fifo_tb.v`

Stimulus from `+test1` / `+test2`, fault hooks for memory, pointer, and flag injections.

FIFO DUT `fifo_sm.v`

Main control path computes `DoRead/DoWrite` and aggregates global `Error` output.

Redundant Safety Monitors

Duplicate FLAGS and pointer counters (`FL_IF/SM`, `RP_IF/SM`, `WP_IF/SM`) for compare-based detection.

SDPRAM_TOP + ECC

`ECC_8BIT_ENC` -> encoded RAM word -> `ECC_8BIT_DEC` with syndrome-based error detect/correct.

Error Consolidation

`Error = FlagError | ReadError | WriteError | EccError` to indicate safety violations.

2) Fault Injection and Simulation Pipeline

Compile

Build RTL and testbench with safety defines from `rtl.f` using VCS.

Campaign Setup

`COLLATERAL/FCM/fcm.tcl` creates campaign with SFF fault model and DUT target.

Fault Simulation

Run `fsim` with permanent PORT faults across testcases.

Coverage Report

Generate `fsim_v.rpt` containing status/location for all injected faults.

Python Safety Analysis

`analyze_report.py` + `fusa_config.json` -> mapping summary, block DC, latent risk outputs.

Optional path: FM waveform dump for fault ID 130 (`fm_fid130_test1.fsdb`) for debug traceability.
